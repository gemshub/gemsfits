GEMSFITS Code for GEM Input Parameter Optimization

About gfshell GUI (Graphical User Interface)

A graphical user interface (GUI) gfshell2 is designed to assist the user throughout the whole workflow of parameter optimization. The GUI also offers a detailed help functionality on how to maintain the experimental database, edit and prepare the gemsfit2 task specification (files), run gemsfit2 program, and explore, save, and plot the fitting results.

Contents

  • Running modes

1. Running modes:

Database Mode - the experimental records from the project database can be viewed in a list and edited in JSON format (Figure 4). Bulk data can be backed up and restored form JSON format files or imported and exported from pre-formatted spreadsheets files previously saved in csv format. The database selection queries can be added and edited in the “Search Query Editor”. The record (sample) list resulted by executing the search query can be then easily exported into the task specification with help of a menu command,  to be used during the fitting run.

Fit Task Mode -  fitting task specifications can be edited, saved to database, or new tasks created from them, and they can all be exported to task input files and executed in gemsfit2 for parameter optimization.

View Results Mode - the results can be imported from gemsfit2 output csv files, viewed and plotted.
