GEMSFITS Code for GEM Input Parameter Optimization

About GEMSFITS code package

Contents

GEMSFITS code package is a tool for fitting internally consistent input parameters of chemical thermodynamic models against the experimental data, and for performing various inverse modelling tasks.

GEMSFITS contains the GEMS3K chemical solver – the Gibbs energy minimization kernel of GEM Software (GEMS) - in which a comprehensive suite of (non)ideal mixing models, equations of state and activity models is implemented.

GEMSFITS allows simultaneous fitting of multiple type of parameters to may types of experimental data.

GEMSFITS can manage and query large amounts of experimental data accumulated in database files in the industry-winning NoSQL BSON format (Binary JSON - JavaScript Object Notation). The data can be imported into the database from reformatted spreadsheets (in csv format), or added/edited directly using the gfshell (Graphical User Interface) ejdb console.

GEMSFITS can also perform statistical evaluation of the fitted parameters (basic statistics, confidence intervals, Monte Carlo simulations, etc.), thus helping the user to assess the goodness of fit.


The GEMSFITS software package consists of several main components:

   1. The gfshell GUI with graphics widget and help viewer helps the user make the fitting process efficient and user-friendly by keeping track of it through task-definition and task-results records in the database, and providing text editors and graphics for input- and results data.

   2. The gemsfit2 parameter optimizer code that reads the experimental data from database and executes a “fitting task” as described in the task configuration file (exported from task definition database record, or edited separately), writing results in a set of csv-format output files

   3. The GEMS3K chemical solver (including TSolMod library of models of mixing) embedded into the gemsfit2 code, which reads chemical system definition from the set of GEMS3K input files provided with the task data, and calculates chemical equilibria and/or phase activity models whenever asked by the parameter optimizer.

   4. The NoSQL experimental database (collection of documents describing experimental samples, fitting task definitions, and fitting task results), with tools for the EJDB database management (http://ejdb.org).


Data type to compare - phase chemical (elements) composition, given in molal or loga (meaning log) units; phase dependent components amount/fraction; phase properties: pH, Eh, amount, volume, RHO, alkalinity, surface area;

Parameters to fit - "G0" (Gibbs free energy at 25 C and 1 bar), "bIC" (bulk composition), "PMc" (interaction parameter coefficients for non-ideal solutions), "DMc", "fDQF", "TK", "P".
