1. -init mode

How to produce a template GEMSFIT configuration file from GEMS3K I/O files (DK DM SD 10.10.2013)

    1. To produce a gemsfit2 input file the user has to run gemsfit2 (command line) in the "-init" mode

>$ gemsfit2 -init <path to <GEMS3K>-dat.lst>

Example: >$ gemsfit -init ./GEMS3K-files/AllDa3T-fwd-dat.lst

<path to <GEMS3K>-dat.lst> is a path to the list of GEMS3K DCH, IPM and DBR input files as exported from the GEM-Selektor code package.

In this mode, GEMSFIT will generate a template input configuration file. The generated GEMSFIT input configuration file will be placed in the directory of gemsfit executable and will be called gemsfit_input.dat (if it already exists will as the user to overwrite or rename). This file should be edited by the user to indicate: path to the database and GEMS3K exported files (1) what experimental data to use (DataSelect), (2) what GEMS run output to compare with the experimental data (DataTarget), (3) mark what parameters to adjust, (4) fitting algorithm and statistics related parameters.
    

2. DataSelect

DataSelect represents a JSON format object used as a query for obtaining the experimental data from the database.

Options:
      '{ ... }': object in JSON format (in braces) describing what to select.
      "sample": [...]: list of comma-separated names of samples,
         or empty string "" to select all samples;
      "usedataset": [...]: list of comma-separated names of experimental datasets to be used,
         or empty string "" to select all available datasets;
      "skipdataset": [...]: list of comma-separated names of experimental datasets to be skipped,
         or empty string "" to not skipp (in this case usedataset will apply);
      "sT": [...]: list of comma-separated minimum and maximum temperature, C,
         or empty string "" to select data for all available temperatures;
      "sP": [...]: list of comma-separated minimum and maximum pressures, bar,
         or empty string "" to select data for all available pressures; 

Example:
      '{"sample" : "",
        "usedataset" : ["CH04", "CH04D"],
        "skipdataset" : ["CH03", "CH03D"],
        "sT" : [100, 1000],
        "sP" : [1, 2500]}'

3. DataTarget

DataTarget represents a JSON format object used to describe with GEMS output data to compare with experimental data.

Options:
    '{ ... }' : object in JSON format (in braces) describing what to compare and which type of function should be minimized.
    "Target":  name of the target (objective) function (OPTIONAL)
    "TT":  type of target function as sum of following terms
       Options:
         "lsq":  w*(measured-simulated)^2;
         "lsq-norm":  w*(measured/avgm-simulated/avgm)^2;
    "WT":  weighting scheme
        Options:
           "inverr": w=1/error;
           "inverr2": w=1/error^2;
           "inverr3": w=1/measured^2;
           "inverr_norm": w=1/(error/<meas(average)>)^2;
    "OFUN":  objective function, a list [] of terms {} for measured properties to compare with their computed counterparts.
     Each term can contain property-value pairs:
       "EPH": for what phase from the experiments
       "EN":  for what element from the phase
       "EP":  for what property of the phase
       "unit":  what unit the values should are given in (overrides the units given in the database for this entry) (OPTIONAL):
          "molal":  mol/(kg H2O), "loga": log(molal), "-loga": negated log(molal);
          "g"; "kg"; "cm3"; "m3"; "molratio": mole ratio
           ...
       "DC":  used only if "CT" is "DC", represents the name of the dependnt component property: Q for amount.
    
The compare options are:
       aqueous phase ("aq_gen") elemental composition in "molal" or "loga"
       aqueous phase ("aq_gen") properties ("EP"): "pH" in "-loga" (or "molal" - molality concentration of H+); "pQnt" - mass in "g" or "kg" 
       other phases composition as element bulk phase concentration in moles ("mol") or to /Si molar ratio ("Simolfrac")
       other phases properties ("EP"): "Q" - mass in "g" or "kg"; "pV" - volume in "cm3" or "m3"; "Eh" - volume in "Volts"
       dependent components ("DC") properties: "Q" - amount in "molfrac"; "@coef" - activity coefictient


Example:
  '{ "Target": "name", "TT": "lsq", "WT": "inverr", "OFUN":
      [
         { "EPH": "aq_gen", "CT": "IC", "CN": "Si", "unit": "molal" },
         { "EPH": "aq_gen", "CT": "IC", "CN": "Al", "unit": "molal" },
         { "EPH": "aq_gen", "CT": "property", "CN": "pH",  "unit": "-loga" }
         { "EPH": "aq_gen", "CT": "DC", "CN": "SiO2@", "DCP": "Q", "unit": "molal" },
      ]
   }'

4. Parameters to fit

GEM input parameters to be fitted can be marked by preceding them with the F, L, and R letter  in <TKval>, <Pval>, <G0>,  <PMc>, <DMc>, <fDQF>, <bIC> ... data object entries in the input file. With F you can mark any type of parameters, with L bIC, and with R G0's which the user wants to have them ase reaction constrined.
Mark with F the parameter value you want to fit independently of other parameters. Options:
    F<initial value> : the initial value with default (10%) upper- and lower boundaries,  example for fitting G0(298) of SiO2@ : F-833444
    F{"IV":<initval>, "LB":<lowbound>, "UB":<upperbound> }
    F{ "IV" :-833444.00006792, "UB" : -800000, "LB" : -900000 }
    Here,  "IV": initial value;
               "UB": upper boundary;
               "LB": lower bundary;
Mark with R the G0(298) value of a dependent component, which depends on G0 of other dependent components via a reaction constraint, as in the following template:
    R{ "IV" : -36819, "Ref" : "SUPCRT92", "nC" : 4, "DC" : "KOH@", "RC" : [ "K+", "H2O@", "H+", "KOH@" ], "Rcoef" : [ -1, -1, 1, 1 ] }
    Here,  "IV": initial value;
               "Ref": bibliographic reference;
               "logK": reaction equilibrium constant at 298 K, 1 bar;
               "nC": numer of components (species) involved in the reaction;
               "DC": name of dependent component whose properties are constrained with this reaction;
               "RC": list [ ] of names of all components (species) involved in the reaction (comma-separated);
               "Rcoef": array [ ] of reaction stoichiometry coeficients (comma-separated), in the same order as in the "RC" list.
Mark with L the bIC (element bulk composition) value of an independent component, which depends on bIC of other elements via a titration constraint similar to the following template in place of the bIC value, and edit it to make the desired changes:
    L{ "LE" :"H", "IV" :113.016746705914, "LEs" :["S", "Cl"], "Lcoef" :[2,1]}
    Here, H (hydrogen) is linked to S (sufur) and Cl (clorine) by titration of H2SO4 and HCl, with the stoechiometric coeficients of 2 and 1, respectively.
               "LE": linked element
               "IV": initial value
               "LEs": the elements linked to
               "Lcoef": linking coeficients (stoechiometric coeficients)

5. Optimization algorithm and statistics

Gemsfit uses NLopt optimization algorithm. The implemented algorithms are: GN_ISRES, GN_ORIG_DIRECT, LN_COBYLA, LN_BOBYQA, LD_MMA, LD_SLSQP, GD_MLSL.

The user can override the previously assigned abounds by giving a general bound percentage in OptBundPerc section. The stoping criterion and maximum number of evaluations can be also set here. In the statistics section the user can specifi the number of points for the sensitivity evaluation, the gradient and sensitivities central difference step.


To run GEMSFIT, a path to the the input configuration file should be given in the command line:

>$ gemsfit -run <path to GEMSFIT input config file>

Example: >$ gemsfit -run ./SS_INPUT/SS_GEMSFIT_AllDa3T_input.dat

The GEMSFIT output files will be written into the "output_GEMSFIT" and the "results_GEMSFIT" subfolders. The output_GEMSFIT contains the log file and the output statistics of the current run. This folder gets DELETED before each run!! The results_GEMSFIT folder contains a comma separated file that stores the calculated, measured, residuals, and fitted parameters for all runs. The file is appended until the user changes the name, moves or deletes it.
