How to maintain experimental databases using EJDB console

Contents

EJDB Command line interface (CLI) to work with GEMSFIT2 database of experimental data

INSTALLATION

Go to http://nodejs.org/ - download the current version, extract it, go to the folder and install it with the following commands:
Unix/Macintosh:

    ./configure
    make
    make install

Install the EJDB CLI with the following command: npm install -g ejdb

The usage and examples for the command line interface can be found on this web page, with its contents copied below with modifications:
https://github.com/Softmotions/ejdb/wiki/EJDB-Command-line-interface

Running EJDB CLI: 
EJDB CLI v1.0.39
usage: ejdb [options] [dbfile]
options:
	-h --help	show this help tip
	-n --no-colors	do not use colored output
	-q --quiet	run in quiet output mode
	-c --cmd	run specified javascript command

$ ejdb
Welcome to EJDB CLI v1.0.39
ejdb>

This is javascript interpreter with db and EJDB objects in the context, to describe db just type: db <ENTER>

ejdb> db
{ open: [Function] (dbFile, [openMode]) Open database,
  status: [Function] Get current database status,
  close: [Function] Close database }
ejdb>

Let's open/create db: 

ejdb> db.open('mydb');
{ file: '/home/adam/mydb', collections: [] }

For opened database we have more methods to operate: 

ejdb> db
{ close: [Function] Close database,
  status: [Function] Get current database status,

  removeCollection: [Function] (cname, [prune], [cb]) Removes collection, if `prune` is true collection db will be erased from disk.,
  save: [Function] (cname, object|array of object, [opts], [cb]) Save/update specified JSON objects in the collection.,

  findOne: [Function] (cname, [qobj], [qobjarr], [hints], [cb]) Retrive one object from the collection,

ejdb>

Then save something: 

ejdb> db.save('mycollection', {foo : 'bar'});
[ { foo: 'bar',
    _id: '50d2f2fa7075995500000000' } ]
ejdb> db.save('mycollection', {foo : 'bar2'});
[ { foo: 'bar2',
    _id: '50d2f3007075995500000001' } ]
ejdb> db.save('mycollection', {foo : 'bar3'});
[ { foo: 'bar3',
    _id: '50d2f3037075995500000002' } ]
ejdb>
Show db status and its collections: 
ejdb> db.getDBMeta()
{ file: '/home/adam/mydb',
  collections: 
   [ { name: 'mycollection',
       file: '/home/adam/mydb_mycollection',
       records: 3,
       options: 
        { buckets: 131071,
          cachedrecords: 0,
          large: false,
          compressed: false },
       indexes: [] } ] }
ejdb>
Let's select first record: 
ejdb> db.findOne('mycollection');
{ _id: '50d2f2fa7075995500000000',
  foo: 'bar' }
Want more: 
var cursor = db.find('mycollection');
Found 3 records
{ _id: '50d2f2fa7075995500000000',
  foo: 'bar' }
{ _id: '50d2f3007075995500000001',
  foo: 'bar2' }
{ _id: '50d2f3037075995500000002',
  foo: 'bar3' }
We can do more with cursor saved in previous step: 
ejdb> cursor
{ pos: 0,
  length: 3,
  close: [Function: close],
  reset: [Function: reset],
  hasNext: [Function: hasNext],
  next: [Function: next],
  field: [Function: field],
  object: [Function: object] }
ejdb> 
ejdb> while (cursor.next()) console.log("foo" + cursor.field("foo"));
foobar
foobar2
foobar3
undefined
ejdb>
Finish this little demo: 
ejdb> db.close();
undefined
ejdb> Bye!
To stop EJDB CLI, press Ctrl-C several times.
