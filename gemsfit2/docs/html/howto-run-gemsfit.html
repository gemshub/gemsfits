How to run GEMSFIT2 program

gemsfit2 is the main tool used for processing/fitting the data. Once the user produced a database file (EJFB format) and exported the GEMS3K chemical system files, he can proceed with the preparation of the fitting routine and start the actual fitting.

gemsfit2 has two running modes: -init and -run.

1. -init

If the user is running gemsfit for the first time or he is using a different database of experiments, different chemical system, or an updated database, he has to first run the program in the "-init" mode. For every change in the experimental database and the GEMS3K exported system files gemsift2 should first be run in the "-init" mode!
Runing in -init mode using the command line needs two arguments:

    $ gemsfit2 -init <path to <GEMS3K>-dat.lst>
    $ gemsfit2 -init ./GEMS-files/mySystem-dat.lst

the second argument is the path to the .lst file that sits in the same folder with the rest of the GEMS3K exported system files. After running gemsfit2 in "-init" mode, a template input file named gemsfit_input.dat is produced using some parts of GEMS3K input files.

2. Editing the input file

The user has to carefully look through and edit the template file according to the instructions (comments) given in the file for each section.
The main options to specify are:
  •   where is the database file
  •   where are the GEMS3K system files
  •   what experimental data to select from the database
  •   what data entries to compare within the objective function
  •   what parameters to fit and how to fit them (independent, reaction, linked), what are initial parameter values and bounds
  •   what fitting algorithm to use (type of algorithm, etc.)
  •   what statistics parameters to compute

3. -run

Run gemsfit2 fitting task using the "-run" mode. Running in -run mode using command line needs two arguments:
   
    $ gemsfi2 -run <path to gemsfit_input.dat>
    $ gemsfi2 -run ./gemsfit_input.dat

4. Output and results

If there were no errors returned by the program after the executing time (which can vary depending on the number of experiments and the number of parameters to fit), the results are written in two places:
   /output - the actual run result statistics file and the run log file are here.This folder is emptied before every run of gemsfit2 in the "-run" mode. It is IMPORTANT to move or save the files elsewhere before making another run of the program!
   /results - this folder is never emptied and contains the csv file fit_results.csv. The file contains the mesured and calculated values, the residuals, and the result parameters for every run. This file can be deleted or   moved by the user - a new file will be automatically created for future runs. The file will be appended until the name is changed, the file is moved or deleted.
